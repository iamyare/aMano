### Defina los siguientes términos: datos, base de datos, DBMS, sistema de bases de datos, catálogo de la base de datos, independencia programa-datos, vista de usuario, DBA, usuario final, transacción enlatada, sistema de bases de datos deductivo, objeto persistente, metadatos y aplicación de procesamiento de transacciones.

- Datos: Son hechos o información que se pueden almacenar y procesar en un sistema informático.
- Base de datos: Es una colección organizada de datos que se pueden acceder, gestionar y actualizar de manera eficiente.
- DBMS: Es el acrónimo de Sistema de Gestión de Bases de Datos (Database Management System), que es un software que permite a los usuarios crear, acceder y gestionar bases de datos.
- Sistema de bases de datos: Es un conjunto de componentes de hardware y software que trabajan juntos para permitir la creación, gestión y uso de bases de datos.
- Catálogo de la base de datos: Es una parte del DBMS que contiene información sobre la estructura de la base de datos y sus restricciones.
- Independencia programa-datos: Es la capacidad de cambiar la estructura de la base de datos sin tener que modificar los programas que acceden a ella.
- Vista de usuario: Es una representación de una parte de la base de datos que se muestra a un usuario final.
- DBA: Es el acrónimo de Administrador de Bases de Datos (Database Administrator), que es la persona responsable de la gestión y mantenimiento de una base de datos.
- Usuario final: Es la persona que utiliza la base de datos para acceder a la información que necesita.
- Transacción enlatada: Es una transacción que se compone de varias operaciones que se ejecutan como una sola unidad.
- Sistema de bases de datos deductivo: Es un tipo de sistema de bases de datos que utiliza la lógica deductiva para inferir nuevos hechos a partir de los datos existentes.
- Objeto persistente: Es un objeto que se almacena en una base de datos y mantiene su estado incluso después de que el programa que lo creó haya finalizado.
- Metadatos: Son datos que describen la estructura de la base de datos y sus restricciones.
- Aplicación de procesamiento de transacciones: Es un software que se utiliza para gestionar y procesar transacciones en una base de datos.

### ¿Qué cuatro tipos de acciones implican bases de datos? Explique brevemente cada uno de ellos.

Los cuatro tipos de acciones que implican bases de datos son:

1. Definición: Implica la especificación de la estructura y los tipos de datos que se almacenarán en la base de datos, así como las restricciones que se aplicarán a los datos.
2. Construcción: Implica la creación física de la base de datos en el sistema informático, incluyendo la asignación de espacio en disco y la creación de tablas y otros objetos.
3. Manipulación: Implica el acceso y modificación de los datos almacenados en la base de datos, como agregar, actualizar o eliminar registros.
4. Compartición: Implica el acceso simultáneo a la base de datos por parte de múltiples usuarios o aplicaciones, lo que requiere mecanismos para garantizar la integridad y consistencia de los datos durante las operaciones concurrent
